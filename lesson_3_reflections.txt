Q) When would you want to use a remote repository rather than keeping all your work
local?
Answer: Here are some possible scenarios: 
	a) When I want to share my work with other colleagues
	b) When I want to send to a server for CI related activities
	c) When I want to make a backup of my code online

Q) Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

Answer: There could be many reasons but for me, it seems logical that I only sync when I have dfinished my section of the work before pulling any new changes. Otherwise, I might encounter some conflicts with my existing work.
